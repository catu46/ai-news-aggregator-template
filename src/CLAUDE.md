# src/ — the project's Python package

This folder is the entire codebase: a personal AI news/repos aggregator. The idea is simple: sources (Reddit, GitHub, X) dump posts into a Postgres database (Supabase + pgvector), each post gets an embedding (Voyage) and a quality verdict from an LLM curator (Haiku 4.5), and whatever passes is delivered on Telegram with 👍/👎 buttons. The votes become a taste signal and an archive searchable by meaning. Here at the package root live only the two data-side "entry scripts" (`pipeline.py` and `seed.py`); each subfolder is a piece of the system with its own CLAUDE.md (read the subfolder's one before touching it). Comments in the code are in English, identifiers in English.

## Files at this root

- `pipeline.py` -> Orchestrates **one complete cycle** of data: `ingest -> embed -> curate`, in that order (each stage feeds the next). Runs with `python -m src.pipeline` and is built to run on a schedule (e.g.: cron on Railway): it is idempotent, each stage only touches what is pending in the database, and deduplication by `(source_platform, source_id)` happens on upsert. **Ingest** builds the per-user sources from `config/sources.yaml` (Reddit, GitHub, X) — the active directions from `/foco` inject topics straight into ingestion, not just reorder. **Embed** vectorizes posts without an embedding in batches. **Curate** asks Haiku for a GLOBAL quality verdict (`similarity_signal=None`, no user signal) and respects the monthly spend cap (`BudgetExceeded` ends the stage gracefully). Failures of a source/post are isolated and logged, they do not bring down the cycle. **Important:** delivery to Telegram does NOT happen here — it is the bot's JobQueue that delivers.

- `seed.py` -> **Cold-start:** loads taste examples from `config/seeds.yaml` BEFORE any real ingestion, so the system already has signal on day 1. Runs with `python -m src.seed`. For each example, it creates a post with `source_platform='seed'` and a deterministic `source_id` (`seed-<sha1>`, so re-running does not duplicate), embeds the text, and writes a pre-loaded vote: `gold => +1` (👍), `noise => -1` (👎), with `origin='seed'`. These votes+embeddings feed RECALL and the affinity prior, but seeds **do not pass through the curator** (verdict is left NULL on purpose — they are taste labels, not approved/rejected posts) and do not enter the delivery flow. (The CHECK on `posts.source_platform` in `db/schema.sql` already includes `'seed'`, `'github'` and `'manual'`, so there is no extra schema step.)

## Subpackages (each has its own CLAUDE.md)

- `common/` -> Core shared by EVERYTHING else. `config.py` (reads `.env` + `config/sources.yaml` + `config/seeds.yaml` into typed objects, e.g.: `Settings`, `UserSources`, `SeedExample`), `db.py` (async access to Postgres/Supabase with pgvector via asyncpg — `upsert_post`, `set_embedding`, `mark_curation`, `record_vote`, `active_focus`, semantic search, etc.), `embeddings.py` (Voyage `voyage-4-lite` wrapper, 1024-dim L2-normalized vectors), `models.py` (shared data models: `IngestedPost`, `Verdict`). See `common/CLAUDE.md`.

- `ingestion/` -> The content **sources**. Every source inherits from `IngestionSource` (`base.py`) and implements `async fetch() -> list[IngestedPost]`; dedup is left to the database. Implementations: `reddit_source.py` (public RSS feed with multireddit), `github_source.py` (trending repos by topic via Search API + README), `x_source.py` (X/Twitter via `twitter-cli` with cookies). `pipeline.run_ingestion` is what instantiates and runs these sources. See `ingestion/CLAUDE.md`.

- `curation/` -> The **curator** and chat-based steering. `curator.py` (the `Curator` ABC interface + the default `AnthropicCurator` using Haiku 4.5 with Structured Outputs and a rubric in a cached system block; `BudgetExceeded` at the spend cap — it is "swappable", switching providers is just reimplementing `classify`), `prompt.py` (the long, static `RUBRIC` that triggers prompt caching), `steering.py` (classifies a chat message from the owner into an intent: `steer` / `recall` / `balance` / `other`). See `curation/CLAUDE.md`.

- `bot/` -> The **Telegram bot** (`bot.py`, python-telegram-bot, long-polling). Locks onto an allowlist of `telegram_user_id` coming from `sources.yaml` (multi-user-ready), delivers approved posts not yet delivered (`deliver_pending`, via JobQueue once a day in 2 buckets and on demand with `/feed`), records 👍/👎 votes from the inline buttons, routes free-form chat (steer/recall/balance) and does semantic search over the archive (`/buscar`). State (Database + Embedder + allowlist) lives in `application.bot_data`. Run with `python -m src.bot.bot`. See `bot/CLAUDE.md`.

- `mcp_server/` -> **MCP** server (`server.py`) that exposes YOUR curated archive as tools for Claude (Code/Desktop) — a thin shell over the `Database`, reusing what the bot already uses (`search_pool` / `recall_voted` / `active_focus`). It lets you ask "what did I like about RAG?" straight from Claude, without opening Telegram. Each person runs their own instance (their Supabase, their `telegram_user_id`), the same multi-tenant isolation as the rest. See `mcp_server/CLAUDE.md`.
